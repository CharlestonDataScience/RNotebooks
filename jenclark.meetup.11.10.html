trainControl

  • pro: quick, painless, ready to go in a few minutes
  • pro: highly customizable
  • pro: mistakes and leaks are easy to spot
  • con: ?

trainControl code

#downSample to create a balanced problem
data <- downSample(Alzheimers, Alzheimers$response)

#create a vector with predictors
y <- data$response

#drop predictor variable from dataset
data <- select(data, -response,-Class, -gender, -Genotype)

#set trainControl with customizable arguments
cv <- trainControl(method='cv',verboseIter=T, classProbs=TRUE)
#method boot, repeatedcv, repeated training/test splits, etc.
#verboseIter prints a training log
#classProbs class probablities are computed for 
#classification models in each resample

caret models, including xgBoost

  • pro: large variety of models (https://topepo.github.io/caret/modelList.html)
  • pro: test a handful all at once and compare
  • con: depending on tuning parameters, can take a while to run
  • focus here on eXtreme Gradient Boosting = xgbTree or xgbLinear
    • (Stochastic Gradient Boosting = gbm & Random Forest = rf)
    • supports regression, classification, and ranking
    • it's fast & accepts sparse input
    • http://xgboost.readthedocs.io/en/latest/R-package/xgboostPresentation.html

caret modeling using train

models <- list()
models[['rf']]<- train(data, y, method='rf', 
                  trControl=cv, tuneLength=3)
models[['rpart']] <- train(data, y, method='rpart',
                  trControl=cv, tuneLength=10)
models[['gbm']] <- train(data, y, method='gbm', 
                  trControl=cv, tuneLength=10)
models[['xgboost']] <- train(data, y, method='xgbTree', 
                  trControl=cv, tuneLength=10)
models[['earth']] <- train(data, y, method='earth', 
                  trControl=cv, tuneLength=3)

let's check our accuracy

accuracy <- names(models) %>%          # For each model name,
  lapply(function(model.name) {        # apply a function that will use 
                                        #that name to
    models[[model.name]]$resample %>%  # fetch a data frame 
                                      #containing cross validation accuracy,
      mutate(Model=model.name)         # attach the model name to 
                                      #that data frame,
  }) %>% do.call(rbind, .)            # and then merge results 
                                      #for all models together
#credit: Eric Czech

Partial Dependence

  • pro: INTERPRETABILITY
  • pro: understand new relationships in the data
  • con: can be misleading with higher-order hidden interactions
  • pro/con: more difficult to reduce down to a sound-bite
